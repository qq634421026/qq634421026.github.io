Java反射机制
  • 对于任意一个类，都能够知道这个类的所有属性和方法；
  • 对于任意一个对象，都能够调用它的任意一个方法和属性;
  • 获取Class对象
    • 每创建编译一个类，JVM就会创建一个对应的Class的实例，并且这个Class对象保存在同名的.class文件里。当我们new一个新对象或者引用静态成员变量时，java虚拟机中的类加载器系统会将对用Class对象加载到JVM中，然后JVM在根据这个类型信息相关的Class对象创建我们需要实例对象或者提供静态变量类型的引用值
    • Class类实例保存了相关类的类型信息，包括属性，方法，构造方法什么的，通过这么Class实例，我们可以在运行时访问对象的属性和方法等。另外通过Class类还可以创建出一个新的对象，这就是反射的基础
    • 获取Class方法
      • Objcet.getClass() byte[] bytes = new byte[1024]; Class c = bytes.getClass();但是基本类型无法使用这种方法
      • XXX.Class Class c = boolean.class; Class c = String.class 基本类型也可以
      • Class.forName() Class c = Class.forName("java.lang.String");
        • Class cDoubleArray = Class.forName("[D");    //相当于double[].class
        • Class cStringArray = Class.forName("[[Ljava.lang.String;");   //相当于String[][].class
      • Class.getSuperclass()获得给定类的父类 Class
    • 通过Class获取修饰符和类型
      • 获取类名 c.getCanonicalName()
      • 获取限定符 Modifier.toString(c.getModifiers())
      • 获取泛型  TypeVariable[] tv = c.getTypeParameters(); tv.getName()
      • 获取所有实现接口  Type[] intfs = c.getGenericInterfaces(); intf.toString
      • 获取父类 c.getSuperclass()
      • 获取所有注解  Annotation[] ann = c.getAnnotations();
  • Member
    • 三个实现类
      • Fieldd 对应类变量
        • getDeclaredField(String name) 获取指定的变量（只要是声明的变量都能获得，包括 private）
        • getField(String name) 获取指定的变量（只能获得 public 的）
        • getDeclaredFields() 获取所有声明的变量（包括 private）
        • getFields() 获取所有的 public 变量
        • 获取变量名称 f.getName()
        • 获取变量类型 f.getType()
        • 获取变量修饰符 Modifier.toString(f.getModifiers())
        • 获取变量注解    Annotation[] ann = f.getAnnotations(); an.toString()
        • 获取变量的值
          • Cat cat = new Cat("Tom", 2);
          • Class c = cat.getClass();
          • fieldName.setAccessible(true);//私有的要取消java语言访问权限
          • Field fieldName = c.getDeclaredField("name");
          • //取值   String name = (String) fieldName.get(cat);
          • //设值  fieldName.set(cat, "Timmy");
      • Method 对应类方法
        • getDeclaredMethod(String name, Class<?>... parameterTypes) 根据方法名获得指定的方法， 参数 name 为方法名，参数 parameterTypes 为方法的参数类型，如 getDeclaredMethod(“eat”, String.class)
        • getMethod(String name, Class<?>... parameterTypes) 根据方法名获取指定的 public 方法，其它同上
        • getDeclaredMethods() 获取所有声明的方法
        • getMethods() 获取所有的 public 方法
        • 获取方法返回类型
          • getReturnType()   获取目标方法返回类型对应的 Class 对象
          • getGenericReturnType()  获取目标方法返回类型对应的 Type 对象
          • 区别
            • 例如 public T function2()，那么各自返回值为：
              • getReturnType() : class java.lang.Object
              • getGenericReturnType() : T
            • 例如public Class<String> function3()，那么各自返回值为：
              • getReturnType() : class java.lang.Class
              • getGenericReturnType() : java.lang.Class<java.lang.String>
        • 获取方法参数类型
          • getParameterTypes() 获取目标方法各参数类型对应的 Class 对象
          • getGenericParameterTypes() 获取目标方法各参数类型对应的 Type 对象
        • 获取方法声明抛出的异常的类型
          • getExceptionTypes() 获取目标方法抛出的异常类型对应的 Class 对象
          • getGenericExceptionTypes()  获取目标方法抛出的异常类型对应的 Type 对象
        • 获取方法的修饰符
          • method.getModifiers();
        • method.isVarArgs() //判断方法参数是否是可变参数
        • 通过反射调用方法
          • //无参数
          • Method sleep = c.getDeclaredMethod("sleep");
          • sleep.setAccessible(true) 
          • sleep.invoke(cat);
          • //有参数
          • Method eat = c.getDeclaredMethod("eat", String.class);
          • eat.invoke(cat, "grass");
          • //不确定参数
          • Class[] argTypes = new Class[] { String[].class };
          • Method varargsEat = c.getDeclaredMethod("eat", argTypes);
          • String[] foods = new String[]{"grass", "meat"};
          • varargsEat.invoke(cat, (Object)foods);
      • Constructor 对应构造函数
        • getDeclaredConstructor(Class<?>... parameterTypes) 获取指定构造函数，参数 parameterTypes 为构造方法的参数类型
        • getConstructor(Class<?>... parameterTypes) 获取指定 public 构造函数，参数 parameterTypes 为构造方法的参数类型
        • getDeclaredConstructors() 获取所有声明的构造方法
        • getConstructors() 获取所有的 public 构造方法
        • 构造方法的名称、限定符、参数、声明的异常等获取方法都与 Method一样
        • 创建对象
          • 1.Class.newInstance()仅可用来调用无参的构造方法；Constructor.newInstance()可以调用任意参数的构造方法。
          • 2.Class.newInstance()会将构造方法中抛出的异常不作处理原样抛出; Constructor.newInstance()会将构造方法中抛出的异常都包装成 InvocationTargetException 抛出。
          • 3.Class.newInstance()需要拥有构造方法的访问权限; Constructor.newInstance()可以通过 setAccessible(true) 方法绕过访问权限访问 private 构造方法。
  • 数组和枚举
    • 数组
      • 数组本质是一个对象，例如int[] intArr数组的类型就是class[I 数组类型中"["的个数代表数组的维度，后面的字母代表数组元素类型
        • 对应
      • 获取一个数组的类型
      • 创建和初始化数组 提供Array类用来创建和初始化数组
      • 创建多维数组
    • 枚举 普通类所能使用的反射方法，枚举都能使用；另外 java 反射额外提供了几个方法为枚举服务。
      • Class.isEnum() 判断还不是枚举类型
  • 缺点
    • 性能开销
      • 反射涉及类型动态解析，所以 JVM 无法对这些代码进行优化。因此，反射操作的效率要比那些非反射操作低得多。我们应该避免在经常被执行的代码或对性能要求很高的程序中使用反射。
    • 安全限制
      • 使用反射技术要求程序必须在一个没有安全限制的环境中运行。
